ThreatLens — Security Architecture (Defence-in-Depth)

5 concentric security layers · OWASP controls · Attack blocking at each layer · Light theme

LAYER 1
🌐 Network / Transport Security
HTTPS enforced (Vercel) TLS certificate auto-renewal Global edge network (47 PoPs) X-Frame-Options: DENY X-Content-Type-Options: nosniff X-XSS-Protection: 1; mode=block Referrer-Policy: strict-origin Permissions-Policy: restricted
❌ Blocks: Cleartext HTTP downgrade attacks ❌ Blocks: Clickjacking (iFrame embedding) ❌ Blocks: MIME-type sniffing attacks
LAYER 2
🚦 Rate Limiting & Quota Protection
4 requests/minute per IP 100 requests/day per IP In-memory sliding window counter 429 Too Many Requests on breach Quota protection for all 6 external APIs
❌ Blocks: DDoS / brute-force flooding ❌ Blocks: API quota exhaustion attacks ❌ Blocks: Credential stuffing enumeration
LAYER 3
🧹 Input Validation & Sanitization
Zod schema enforcement Type enum validation (ip|domain|url|hash) Max 500 character limit sanitizeIOCValue() strips $ Strips < > ' " special chars Regex pattern validation per IOC type 400 Bad Request on schema failure
❌ Blocks: NoSQL Injection ($ operators) ❌ Blocks: XSS via HTML injection ❌ Blocks: SQL-i / JSON injection ❌ Blocks: Oversized payload attacks
LAYER 4
🔐 Authentication & Session Security
JWT HS256 — algorithm explicitly set 24-hour token expiry bcrypt 12 salt rounds (password hashing) SYSTEM_USER_ID fallback for anonymous verifyAuth() middleware on all protected routes Token stored in HttpOnly context
❌ Blocks: JWT algorithm confusion (none/RS256) ❌ Blocks: Token forgery / tampering ❌ Blocks: Weak password rainbow table attacks ❌ Blocks: Unauthorized API access
LAYER 5
🛡️ Data & API Protection
SSRF blocklist in safeFetch() Blocks: 127.0.0.1, 169.254.x.x Blocks: 10.x.x.x, 192.168.x.x ranges MongoDB parameterised queries (Mongoose) Sensitive field redaction in logger.ts 10s fetch timeout — prevents hung requests API keys never logged or exposed to client
❌ Blocks: SSRF to internal AWS metadata ❌ Blocks: NoSQL operator injection via queries ❌ Blocks: API key leakage in logs ❌ Blocks: Internal network probing
🔴 OWASP Top 10 Coverage
A01 Broken Access Control → Layers 4 & 5
A02 Cryptographic Failures → Layer 1 (TLS) + Layer 4 (bcrypt)
A03 Injection (SQLi/NoSQLi/XSS) → Layer 3
A04 Insecure Design → Layers 2, 3, 4
A07 Identification & Auth → Layer 4
A10 SSRF → Layer 5 safeFetch() blocklist
